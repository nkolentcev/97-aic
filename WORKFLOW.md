# Workflow работы с агентами

Этот документ описывает процесс работы с проектом с использованием системы координации ИИ-агентов согласно `ai-template.md`.

## Процесс разработки

### 1. Анализ запроса

При получении задачи:
- Определить, какие компоненты требуются
- Выбрать нужных агентов из списка
- Спланировать последовательность работы

### 2. Делегирование задач

Используйте теги для обращения к конкретным агентам:

#### `<agent_go>` - Backend разработка

Примеры задач:
- Добавление новых API endpoints
- Интеграция с внешними сервисами
- Оптимизация производительности
- Работа с конфигурацией

#### `<agent_svelte>` - Frontend разработка

Примеры задач:
- Создание новых компонентов
- Улучшение UI/UX
- Оптимизация производительности клиента
- Интеграция с новыми API

#### `<agent_devops>` - Инфраструктура

Примеры задач:
- Настройка CI/CD
- Конфигурация деплоя
- Настройка мониторинга
- Оптимизация сборки

### 3. Сборка и проверка

После получения результатов от агентов:
- Проверить совместимость компонентов
- Убедиться в соответствии стилю кода
- Проверить работу интеграций
- Обновить документацию

### 4. Финальный ответ

Предоставить:
- Структурированный код
- Обновленную документацию
- Инструкции по использованию
- Примеры работы

## Примеры workflow

### Добавление новой функции

```
1. <agent_go>
   Добавь endpoint /api/history для получения истории сообщений.
   Используй in-memory хранилище.
   </agent_go>

2. <agent_svelte>
   Создай компонент HistoryPanel для отображения истории.
   Интегрируй с новым endpoint.
   </agent_svelte>

3. <agent_devops>
   Обнови документацию API.md с новым endpoint.
   </agent_devops>
```

### Исправление бага

```
1. Анализ: Определить компонент с багом
2. Делегирование: Обратиться к соответствующему агенту
3. Проверка: Протестировать исправление
4. Документация: Обновить при необходимости
```

## Соглашения

### Формат запросов к агентам

- Четкое описание задачи
- Указание файлов для изменения
- Требования к результату
- Примеры использования (если применимо)

### Формат ответов агентов

- Структурированный код
- Пояснения изменений
- Ссылки на файлы
- Примеры использования

## Интеграция с Git workflow

1. Создать feature ветку: `feature/название`
2. Работать с агентами для реализации
3. Коммитить изменения с понятными сообщениями
4. Создать PR с описанием изменений
5. После merge - автоматический деплой через CI/CD

## Документация

Все изменения должны сопровождаться обновлением документации:
- API изменения → `docs/API.md`
- Архитектурные изменения → `docs/ARCHITECTURE.md`
- Новые функции → `docs/README.md` или соответствующий раздел
