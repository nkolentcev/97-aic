## Обновление документации

Документация обновляется вместе с кодом. При изменении функциональности:

1. Обновите соответствующий раздел документации
2. Проверьте ссылки на другие документы
3. Убедитесь, что примеры кода актуальны
4. Коммитьте изменения вместе с кодом

## Формат документации

- Все документы в формате Markdown
- Код в блоках с указанием языка
- Ссылки на файлы проекта используют относительные пути
- Примеры команд для копирования
