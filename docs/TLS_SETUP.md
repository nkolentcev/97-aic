# Настройка TLS для GigaChat API

GigaChat API использует сертификат НУЦ Минцифры, который может быть не установлен в системе по умолчанию.

## Вариант 1: Установка сертификата НУЦ Минцифры

### Ubuntu/Debian:

1. Скачайте сертификат:
```bash
sudo wget https://www.gosuslugi.ru/crt/rootca.cer -O /usr/local/share/ca-certificates/russian_trusted_root_ca.crt
```

2. Обновите список сертификатов:
```bash
sudo update-ca-certificates
```

3. Перезапустите приложение

### Альтернативный способ:

Скачайте сертификат вручную с [сайта Госуслуг](https://www.gosuslugi.ru/crt) и установите его в систему.

## Вариант 2: Временное отключение проверки сертификата (только для тестирования!)

**ВНИМАНИЕ:** Это небезопасно и должно использоваться только для разработки и тестирования!

Установите переменную окружения перед запуском:

```bash
export GIGACHAT_SKIP_TLS_VERIFY=true
cd backend
go run .
```

Или запустите одной командой:

```bash
GIGACHAT_SKIP_TLS_VERIFY=true go run .
```

## Проверка работы

После установки сертификата или настройки переменной окружения, перезапустите backend и попробуйте отправить сообщение. Ошибка `certificate signed by unknown authority` должна исчезнуть.
